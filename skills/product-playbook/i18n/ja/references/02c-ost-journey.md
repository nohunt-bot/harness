# ステージ1：ディスカバリー — OST + ジャーニーマップ

## 1.4 Opportunity Solution Tree（OST）

**対象：フルモード / 高完成度**

OSTはプロダクト目標からスタートし、機会とソリューションを体系的に結びつけます：

```
[プロダクト目標 / 望むアウトカム]
    │
    ├── [機会1：ユーザーのペインポイントまたはニーズ]
    │       ├── [ソリューション1a]
    │       └── [ソリューション1b]
    ├── [機会2：ユーザーのペインポイントまたはニーズ]
    │       └── [ソリューション2a]
    └── [機会3：ユーザーのペインポイントまたはニーズ]
            └── [ソリューション3a]
```

コア原則：
- 目標（Outcome）は測定可能な結果であり、機能やアウトプットではない
- 機会はユーザーリサーチから来る、社内ブレインストーミングからではない
- ソリューションは機会にマッピングする — 機会をスキップしてソリューションに直行しない
- まず広く、次に深く：まずすべての機会をリスト化し、次に1つずつソリューションを探索

## 1.5 ユーザージャーニーマップ

**対象：フルモード / 高完成度 / オーディエンスがデザイナーの場合**

**ステップ1：概要テーブル**

```
**[ペルソナ名] — タスク：[タスクの説明]**

| ステージ | コア行動 | 感情 | 主要ペインポイント |
|---|---|---|---|
| [ステージ1] | [主要行動の一行説明] | [感情 + 絵文字] | [最も重要なペインポイント] |
```

**ステップ2：各ステージの詳細展開**

```
> **ステージ：[ステージ名]**
> - **行動**：[このステージでユーザーが実際に行うこと]
> - **思考**：[ユーザーの頭の中で考えていること。一人称がベスト]
> - **感情**：[感情状態とその理由]
> - **ステークホルダー**：[このステージに関与する人]
> - **課題**：[具体的な困難やフラストレーション]
```

**ステップ3：グルーピング**
- ステージが細かすぎる場合は、大きなステージグループに統合
- ステージ横断でペインポイントを集約し、コアペインポイントにフラグを付ける
