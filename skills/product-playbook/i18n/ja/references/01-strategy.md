# プロダクト戦略レイヤー

> **注意：各モードで実行するステップは、SKILL.mdの「モードステップ順序」で正式に定義されています。以下は参考情報です。**

**対象：フルモード、高完成度、オーディエンスが経営層/クロスファンクショナル連携の場合**
**スキップ可能：クイックモード、ビルドモード、低完成度の場合**

## Strategy Blocks（Chandra Janakiraman / Headspace / Meta）

良い戦略の階層構造 — 各レイヤーは次のレイヤーの基盤です：

```
Mission（ミッション）
  └→ Vision（ビジョン）— 5-10年後、世界がどうなっていてほしいか？
       └→ Strategy（戦略）— そのビジョンにどうやって到達するか？（重要な選択とトレードオフ）
            └→ Goals / OKRs — 次の6-12ヶ月の優先事項
                 └→ Roadmap（ロードマップ）— 具体的に何を作るか？
                      └→ Tasks（タスク）— 誰が何をいつやるか？
```

## Richard Rumeltの良い戦略の核心

- **Diagnosis（診断）**：直面している課題を明確に定義する（すべての問題ではなく、最も重要な一つ）
- **Guiding Policy（基本方針）**：全体的なアプローチ（目標ではなく、方法論）
- **Coherent Actions（一貫した行動）**：互いに強化し合う具体的な行動。バラバラの計画の寄せ集めではない

> 悪い戦略のサイン：診断なしの壮大な目標；華麗な言葉で中身のない思考を隠す；あらゆる計画を「戦略」と呼ぶ。

## Shreyas Doshiのプロダクトワーク3レベル

プロダクトの問題に取り組む前に、どのレベルで仕事をしているか特定してください：

```
レベル3：Product Excellence — 正しいことを卓越して実行する
レベル2：Product Strategy — 正しいことをやる
レベル1：Product Foundation — 物事をやるための基盤がある（文化、プロセス、人材）
```

> ほとんどのPMはレベル3に時間を費やしすぎ、レベル2の問題を軽視しています。いわゆる「実行の問題」のほとんどは、根本的には戦略の問題です。

## Shreyas DoshiのLNO時間配分フレームワーク

毎週の業務項目について、まず問いかけてください：これはプロダクトにどんなタイプのインパクトを与えるか？

```
L（Leverage）：戦略、ビジョン、文化 → 十分な時間を投資し、卓越性を追求
N（Neutral）：一般的なコラボレーション、定例コミュニケーション → しっかりやるが、完璧を追求しない
O（Overhead）：事務、会議、書類作業 → 素早く終わらせ、過剰投資しない
```

> 節約したO時間を、放置していたL業務に振り向けてください。

## OKRの書き方ガイドライン

Strategy BlocksのGoals/OKRsは、戦略を下位に展開するための重要レイヤーです。良いOKRの最低限のルール：

**Objective**：定性的、インスパイアリング、理解しやすい。達成したい状態を記述し、ToDoではない。
- ✅ 良いO：「新規ユーザーが初日にプロダクトのコア価値を感じられるようにする」
- ❌ 悪いO：「オンボーディングのリデザインを完了する」（それはタスクであり、目標ではない）

**Key Results**：定量的、測定可能、期限付き。目標を達成したかどうかをどう判断するかを記述。
- ✅ 良いKR：「新規ユーザーのD1コアアクション完了率を20%から40%に向上」
- ❌ 悪いKR：「新しいオンボーディングフローをリリース」（それはアウトプットであり、アウトカムではない）

**よくある落とし穴：**
- タスクリストをOKRに偽装（「機能Xを完了」はKRではない）
- OKRが多すぎ（四半期に2-3個のObjective、各Objectiveに3-5個のKRを目安に）
- KRが互いに矛盾、または無関係
- 遅行指標のみで先行指標がない

**例：**
```
O：ターゲットユーザーに当社プロダクトを愛してもらう（PMFレベル2 → レベル3）
  KR1：D28リテンションを12% → 20%に
  KR2：Sean Ellis Scoreを28% → 40%に
  KR3：月間オーガニック紹介シェアを10% → 25%に
```

## プロダクトの3つのコア質問（全プロセスを通じて）

これら3つの質問は順番に回答する必要があります — **順序を入れ替えることはできません**：

> **Q1：どうやって人を呼び込むか？**
> **Q2：どうやってAha Momentにできるだけ早く到達させるか？**
> **Q3：どうやってコア価値を繰り返し提供するか？**

Defineステージでは、これらは以下に変換されます：
- **誰のためか？**
- **なぜ作るのか？**
- **何を作るのか？**
