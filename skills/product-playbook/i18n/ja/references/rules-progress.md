# 🔁 進捗の永続化と中断リカバリー

> ユーザーが「一時停止」「保存」と言った時、またはスキルが起動時に進捗をチェックする時に読み込みます。

## 進捗ファイルフォーマット

各ステップ完了後、プロジェクトディレクトリに`.product-playbook-progress.md`を作成または更新：

```
# Product Playbook Progress Save

- モード：[クイックモード / フルモード / ...]
- プロダクトタイプ：[B2C / B2B / ...]
- プロダクト説明：[ユーザーのプロダクト説明]
- 現在の進捗：S[X] / S[Y]
- 最終更新：[タイムスタンプ]

## 完了ステップ

### S1：[ステップ名] ✅
[このステップのコア出力 — リカバリー時にやり直す必要がないよう十分な詳細を保持]

### S2：[ステップ名] ✅
[同上]

## 未着手ステップ
- S3：[ステップ名]
- S4：[ステップ名]
- ...
```

### 機能拡張モードの例
```markdown
モード：機能拡張
ステップ：S2/S4
S1：問題 + 既存システムのコンテキスト ✅
S2：3つの並行ソリューション + AI推奨 ▶️
S3：リスク評価 ⬜
S4：実行スコープ ⬜
```

## トリガールール

1. **自動保存**：各ステップが完了しユーザーに確認された後、即座に進捗ファイルを更新
2. **起動時チェック**：スキルがトリガーされた際、まず`.product-playbook-progress.md`が存在するか確認。存在する場合、表示：
```
未完了のプロダクト企画進捗を検出しました（[モード名]、S[X]/S[Y]）：
  1️⃣ 続行 — S[X]から再開
  2️⃣ 最初から — 古い進捗をクリアして最初から開始
（1または2を入力）
```
3. **一時停止コマンド**：ユーザーが「一時停止」「他のことをする」「保存」と言った場合、進捗ファイルが更新されたことを確認し回答：「進捗を.product-playbook-progress.mdに保存しました（S[X]/S[Y]）。次回このプロジェクトでスキルを起動する際に自動的に検出されます。」
4. **完了時のクリーンアップ**：フロー全体が完了し最終ドキュメントが作成された後、進捗ファイルを削除するかユーザーに確認
6. **バージョン管理リマインダー**：`.product-playbook-progress.md`が初めて作成された際、ユーザーにリマインド：「⚠️ `.product-playbook-progress.md`を`.gitignore`に追加することをお勧めします — このファイルには機密プロダクト戦略情報が含まれる可能性があります。」
5. **中断保存**：フロー中に無関係なプロンプトが検出された場合（SKILL.mdフロー中断処理ルール参照）、現在のステップが完了していなくても進捗を保存。保存フォーマットで現在のステップには✅ではなく🔶（進行中）を使用し、部分的に作成されたコンテンツを保持：
```
### S[X]：[ステップ名] 🔶（進行中、部分完了）
[部分的に作成されたコンテンツ]
⚠️ このステップは未完了 — リカバリー時はここから再開
```
