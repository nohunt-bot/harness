# ステージ1：ディスカバリー — ペルソナ構築

## Continuous Discovery Habits（Teresa Torres）

1つの重要な習慣を構築してください：**毎週少なくとも1人のターゲットユーザーと話す。** ディスカバリーは一回限りの儀式ではなく、継続的なシステムです。

> "プロダクトディスカバリーは継続的な習慣であるべきで、プロジェクト開始前の一回限りの儀式ではありません。" — Teresa Torres

## 1.1 ペルソナ表の構築

ペルソナは年齢や性別で分類するのではなく、**目的 / タスク / モチベーション**で異なるタイプのユーザーを区別します。

```
| フィールド | ペルソナ1：[ニックネーム] | ペルソナ2：[ニックネーム] | ペルソナ3：[ニックネーム] |
|---|---|---|---|
| 目的 / タスク / モチベーション | | | |
| 規模（SCALE） | | | |
| 課題 / チャレンジ / 動機 | | | |
| 現在のアプローチとその理由 | | | |
| 頻度 | | | |
| 情報源 | | | |
| 導入 / 実行の障壁 | | | |
```

セグメンテーションロジックを説明し、MECE（相互排他的で網羅的）であるか確認し、プライマリTAとセカンダリTAを特定してください。

### 📝 ペルソナ品質チェックリスト
- ✅ セグメンテーションは「目的/タスク/モチベーション」に基づいているか？（デモグラフィックスではなく）
- ✅ ペルソナはMECE（相互排他的でターゲット市場を網羅）か？
- ✅ プライマリTAとセカンダリTAが明確に特定されているか？
- ✅ 各ペルソナの「課題/チャレンジ」は実際の観察または合理的な推論に基づいているか？
- ✅ 「現在のアプローチとその理由」はワークアラウンドを特定できるほど具体的か？
- ❌ よくある問題：年齢/性別によるセグメンテーション、ペルソナ間の差異が最小限、ペインポイントが曖昧すぎる

## 1.2 ペルソナカードの構築

```
## [ペルソナのニックネーム]：[一行の説明]

**基本情報**：年齢 / 性別 / 職業 / 居住地 / 性格特性
**背景**：[プロダクトに関連する背景の説明]
**目標 / タスク**：[目標1]、[目標2]
**現在のアプローチとその理由**：[現在何をしていて、なぜそうしているか]
**情報源**：[関連情報をどこから得ているか]
**障壁 / 課題 / チャレンジ / フラストレーション**：[ペインポイント1]、[ペインポイント2]、[ペインポイント3]
```
