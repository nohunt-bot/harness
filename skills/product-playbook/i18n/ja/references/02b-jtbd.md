# フェーズ1：ディスカバリー — JTBD分析

## 1.3 JTBD（Jobs to Be Done）分析

> "分析の単位は消費者ではなく、消費者が達成しようとしているジョブである。" — Clayton Christensen

**JTBD Statement公式：**
```
[ターゲット顧客] + [どんなジョブコンテキストで] + [どんなジョブ]を達成したい
```

例：住宅ローンの選択肢を比較中の初めてのマイホーム購入者が、銀行に連絡できない深夜に月々の支払額を素早く見積もり、パートナーと資金計画を一緒に確認したい。

**JTBD 4タイプ分析表：**

```
| JTBDタイプ | 定義 | ペルソナ1 | ペルソナ2 |
|-----------|------------|-----------|-----------|
| 機能的ジョブ | 特定のタスクを完了する、または機能的な目標を達成する | | |
| 感情的ジョブ | どう感じたいか、どんな気持ちになりたいか | | |
| 社会的ジョブ | 他者からどう見られたいか | | |
| ジョブコンテキスト | どんな状況でこのジョブを達成する必要があるか | | |
```

**JTBD深掘り5つの質問：**
1. **根本的な問題**：ユーザーが表面的に述べるニーズの背後にある、本当に解決しようとしていることは何か？
2. **現在の制約**：どんなソリューションが特定の制約により排除されたか？
3. **現在のワークアラウンド**：ユーザーは現在どう対処しているか？どんな回避策を構築したか？
4. **ギャップ**：現在のワークアラウンドのどこが不十分か？（このギャップがあなたの機会）
5. **理想のソリューション**：制約が取り除かれた場合、ユーザーの理想のソリューションはどんなものか？

**Teresa Torresユーザーインタビューベストプラクティス：**
- ユーザーの**実際の過去の行動**にフォーカスし、仮定的な将来の行動ではない
- 「この問題に最後に直面した時、どうしましたか？」と聞く。「どんな機能が欲しいですか？」ではない
- 最もよくある間違い：仮定的な質問をする、ソリューションバイアスを持ち込む、詳細を掘り下げない

### 📝 JTBD品質チェックリスト

ClaudeはJTBD出力後にセルフチェックを行う必要があります（各項目に✅または❌を付け、❌の項目には改善方法を記載）：
- [ ] 具体的なコンテキストが含まれているか？（「いつでもどこでも」ではなく、「銀行に連絡できない深夜に」）
- [ ] 単一のコアジョブにフォーカスしているか？（3つのジョブを1文に詰め込んでいないか）
- [ ] 機能的、感情的、社会的ジョブがすべて特定されているか？
- [ ] 「このソリューションは本当にこのジョブに対応しているか？」の評価に使えるか？
- [ ] 「現在のワークアラウンド」と「ギャップ」が含まれているか？（ギャップ＝機会）
- [ ] 深掘り質問のQ5は感情的動機 / 職業的アイデンティティ / 心理的恐怖に到達しているか？（単なる機能的記述ではなく）

**実行ルール（ハードゲート）：**
- 各項目に必ず✅または❌を付ける — 空白の[ ]や説明のない✅リストは不可
- すべての項目が✅の場合、追加で「この分析の最も弱い部分は何で、どう強化するか」を記述する必要がある
- ❌ よくある問題：抽象的すぎる、複数のジョブが統合されている、コンテキストが欠落、プロダクト機能をジョブの記述に置き換えている、Q5が機能レベルにとどまっている

### 🏢 B2Bプロダクト深掘り要件

**B2Bプロダクト（B2B2Cを含む）は以下の分析を完了する必要があります：**

#### 組織レベルのジョブ分析（必須 — 少なくとも2レベルをカバー）

| レベル | 説明 | 例 |
|-------|-------------|----------|
| **戦略的ジョブ** | 組織/経営レベルの部門横断的ニーズ | コンプライアンス監査、コスト管理、人材最適化 |
| **運用的ジョブ** | プロセス/部門マネージャーレベルの調整ニーズ | 承認ワークフロー管理、チーム間情報同期 |
| **タスクジョブ** | 個々のユーザーの日常業務ニーズ | フォーム記入、ステータス確認、レポート出力 |

#### 購入者 vs. ユーザー分析（必須）

購入者とユーザーが異なる人物の場合、それぞれのJTBDを個別に分析：
- **購入者のジョブ**：購買決定に影響するジョブ（ROI正当化、リスク低減、コンプライアンス要件）
- **ユーザーのジョブ**：日常業務中に達成すべきジョブ（効率向上、エラー削減）
- 同一人物の場合、「このシナリオで意思決定者が同時にユーザーである理由」を説明

#### 深掘り5つの質問 — B2B強化版

**Q5は少なくとも以下のレベルの1つに到達する必要があります**（例）：
- ✅ 職業的アイデンティティ：「彼女は経営陣の前で無能に見えることを恐れている。このレポートが部門の信頼性を代表するから」
- ✅ 感情的動機：「彼は直属の部下に、数字を完全に把握していることを示したい」
- ✅ 心理的恐怖：「彼女の最大の恐怖は監査人にプロセスのギャップを発見されること — 以前すでに一度警告を受けている」
- ❌ 不合格例：「より良いツールで効率を改善したい」（機能レベルにとどまっている）

#### 競合代替品分析（必須）

ユーザーが実際に使用している代替品をリスト化：
- 少なくとも2つの名前付き既存ツール（例：Slack / Excel / 紙のフォーム / メール / 口頭連絡）
- 各ツールについて「根本的な欠陥」を説明：機能が弱いのではなく、「この欠陥がなぜ受け入れられ、未解決のまま放置されているか」（組織の慣性？切り替えコスト？経営層が気にしない？）

### 📋 ユーザーインタビュー計画テンプレート

```
## ユーザーインタビュー計画

**リサーチ目標**：[ターゲットペルソナ]が[ジョブコンテキスト]で[特定の問題]にどう対処しているかを理解する
**スクリーニング基準**：
  - 過去[X日/週]以内に[特定の行動]を経験していること
  - 除外：[適さない人 — 例：社内従業員、既知のパワーユーザー]

**コア質問（5-7問）**：
1. 最後に[問題]に直面した時、どのように対処したか手順を教えていただけますか？（行動回想）
2. そのプロセスで最もフラストレーションを感じた、または時間がかかった部分は？（ペインポイント特定）
3. 他のアプローチを試しましたか？なぜ試した/試さなかった？（現在の代替手段）
4. その部分がより良くなるとしたら、「より良い」とはどんな状態ですか？（理想の状態）
5. これはどのくらいの頻度で起きますか？最後はいつでしたか？（頻度と緊急性）
6. あなた以外に、この問題の影響を受ける人はいますか？（ステークホルダーマッピング）
7. 1-10のスケールで、この問題のあなたにとっての深刻度は？その理由は？（ペインの定量化）

**フォローアップ戦略**：
  - インタビュイーが「普段は...」と言った時 → 「具体的に前回はどうなりましたか？」と聞く
  - インタビュイーが感情に言及した時 → 「その気持ちをもう少し具体的に説明いただけますか？」と聞く
  - インタビュイーがツール/方法に言及した時 → 「そのアプローチを選んだ理由は何ですか？」と聞く

**文書化フォーマット**：
  - 逐語録または録音
  - インタビュー後24時間以内に、以下をタグ付け：重要な引用 / ペインポイント / 驚くべき発見 / 仮定との矛盾
```
