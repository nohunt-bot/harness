# 4단계: Deliver — 가설 검증 + 제품 스펙 요약

## 4.5 가설 검증 계획

**적용: 중간/높은 완성도 / 독자가 데이터 사이언티스트/본인인 경우**

```
| 핵심 가설 | 검증 방법 | 검증 시점 | 성공 기준 | 가설이 틀리면... |
|----------|----------|----------|----------|----------------|
| | 알파 사용자 인터뷰 / 데이터 | | | |
```

원칙: 정성적 검증(사용자 인터뷰, 일지 연구) + 정량적 검증(데이터)의 비중은 동등; 자신의 확증 편향에 경계하세요.

## 4.6 제품 스펙 요약

제품 스펙 요약은 모든 모드의 최종 산출물입니다 (퀵 모드는 1페이지 방향 요약을 작성 — 구조는 다르지만 취지는 동일).
제품 스펙 요약은 세 가지 섹션으로, 각기 다른 읽기 깊이에 대응합니다:

### 섹션 1: 의사결정 요약 (30초 내 읽기 가능)

```
**제품 원라이너**: [PR-FAQ 헤드라인]
**대상**: [타겟 Persona 한 문장]
**핵심 가치**: [JTBD 기능적 Job + 감정적 Job, 한 문장]
**North Star Metric**: [지표명 + 정의]
**현재 PMF 수준**: [Level 1-4 + 한 문장 설명]
```

### 섹션 2: 실행 경계 (2분 내 읽기 가능)

```
**핵심 JTBD 선언문**: [타겟 고객] + 이 원하는 것은 [Job] + [Job 맥락]에서
**Aha Moment**: 사용자가 [행동]을 완료하면, 핵심 가치를 경험. 목표: [X] 이내 도달.
**MVP 필수 기능**: [가장 중요한 기능/역량 3-5개]
**명시적 제외 (이 버전에서 의도적으로 배제)**: [명시적 제외 항목 + 이유]
**핵심 가설**: [검증이 필요한 핵심 가설 2-3개]
**Pre-mortem 상위 리스크**: [가장 가능성 높은 실패 원인 2-3개]
**Hero Metric**: [가장 중요한 성공 지표 1-2개 + 목표 값]
```

### 섹션 3: 심층 참조 (필요시 참고)

```
**JTBD 분석**:
  - 기능적 Job:
  - 감정적 Job:
  - 사회적 Job:
**April Dunford 포지셔닝**:
  - 실제 경쟁 대안:
  - 고유 속성:
  - 타겟 사용자를 위한 핵심 가치:
  - 타겟 시장 특성:
  - 시장 카테고리:
**핵심 HMW 질문**: [Define 단계에서]
**솔루션 설명**: [PR-FAQ 요약에서]
**비즈니스 모델**: [수익 모델 + 가격 전략, 완료된 경우]
**GTM 전략**: [초기 획득 채널 + 출시 전략, 완료된 경우]
```

세 가지 섹션을 완료한 후, Claude는 **사용자가 요청하지 않아도 다음 세 가지 섹션을 능동적으로 추가해야 합니다**:

---

### ⚠️ 리스크 레지스터

전체 기획 프로세스에서 식별된 모든 리스크를 통합하세요 (출처: Pre-mortem + 기회 평가 + 가설 검증):

```
| 리스크 유형 | 리스크 설명 | 심각도 (높음/중간/낮음) | 완화 조치 있음? |
|-----------|-----------|---------------------|--------------|
| 시장 리스크 | | | |
| 사용자 가정 리스크 | | | |
| 기술 리스크 | | | |
| 경쟁 리스크 | | | |
| 실행 리스크 | | | |
```

---

### 🔍 갭 및 사각지대

Claude는 전체 대화 플로우를 기반으로 다음 유형의 갭을 능동적으로 식별합니다:

```
| 유형 | 설명 | 권장 다음 단계 |
|------|------|-------------|
| 미실행 단계 | [어떤 프레임워크가 건너뛰어졌고 어떤 사각지대를 유발할 수 있는지] | |
| 불충분한 데이터 | [어떤 결정이 실제 사용자 데이터가 아닌 가정에 기반하는지] | |
| 논리적 비약 | [추론에 중간 근거가 부족한 부분] | |
| 미검증 가설 | [아직 검증 방법이 없는 모든 핵심 가설 나열] | |
| 불명확한 정의 | [실행 전 더 정밀한 정의가 필요한 개념] | |
```

---

### 💡 추가 권장사항

Claude는 제품의 특성, 단계, 알려진 리스크를 기반으로 3-5개의 구체적이고 실행 가능한 권장사항을 제공합니다:

```
1. [최우선]: [구체적 행동] — 이유: [왜 이것이 가장 중요한지]
2. [2순위]: [구체적 행동] — 이유:
3. [3순위]: [구체적 행동] — 이유:
...
```

권장사항은 다음을 포함할 수 있습니다: 사용자 검증 방법, MVP 경계 조정, 경쟁 전략, 지표 설계, 팀 배분, 다음 마일스톤 우선순위.
