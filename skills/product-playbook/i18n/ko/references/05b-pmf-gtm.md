# 4단계: Deliver — PMF + GTM + 비즈니스 모델

## 4.3 Todd Jackson의 (First Round Capital) 4단계 PMF 프레임워크

```
| 단계 | 이름 | 특징 | 해야 할 일 | 전형적 회사 단계 |
|------|------|------|----------|---------------|
| Level 1 | 초기 PMF | 솔루션에 기꺼이 비용을 지불할 고객 3-5명 확보 | 이 초기 사용자를 깊이 서비스하고, 반복 가능한 패턴 찾기 | Pre-seed / Seed |
| Level 2 | 발전하는 PMF | 입소문 시작, 하지만 성장이 아직 불안정 | 확장 가능한 획득 채널 찾기, 리텐션 개선 | Seed / Series A |
| Level 3 | 강력한 PMF | "물고기가 배에 뛰어들어옴" — 추천이 주요 성장 동력 | 확장 가능한 수요 채널 구축, 효율 지표 개선 | Series A / B |
| Level 4 | 극강의 PMF | 지표가 견고, 회사 100명 이상, TAM 확장 고려 | 새로운 시장 진출 또는 새 제품 출시 | Series C+ |
```

**Four P's (Level 1-2에서 PMF를 찾기 위한 핵심 질문):**
1. Problem: 해결하려는 문제가 정말로 충분히 긴급하고 중요한가?
2. Promise: 제품이 사용자에게 어떤 약속을 하는가? 그들이 믿는가?
3. Product: 제품이 실제로 그 약속을 이행하는가?
4. Pitch: 타겟 사용자가 어떻게 제품을 발견하고 믿게 할 것인가?

## 4.7 GTM 전략 (Go-to-Market)

**적용: 풀 모드 / 높은 완성도 / 독자가 경영진/영업/마케팅인 경우**

제품이 만들어진 후, 타겟 사용자가 어떻게 알고 사용하기 시작하게 할 것인가?

### 획득 채널 선택

```
| 채널 유형 | 적합한 경우 | 전형적 방법 | 핵심 지표 |
|----------|-----------|-----------|----------|
| Product-Led Growth (PLG) | 사용자가 셀프서비스로 가치를 경험할 수 있는 제품 | 무료 체험, Freemium, 바이럴 초대 | 가입 전환율, 활성화율, 초대율 |
| 콘텐츠 & 커뮤니티 | 타겟 사용자가 공통 정보 니즈나 커뮤니티를 공유 | 블로그, SEO, 커뮤니티 구축, KOL | 트래픽, 콘텐츠 전환율, 커뮤니티 참여도 |
| 유료 획득 | 빠른 검증이 필요하거나 명확한 ROI | SEM, 소셜 광고, 리타게팅 | CAC, ROAS, LTV/CAC |
| Sales-Led | B2B 고가 ACV, 데모와 협상 필요 | SDR, AE, 데모, 기업 계약 | 파이프라인, 성사율, ACV |
| 파트너십 | 타겟 사용자가 특정 플랫폼이나 생태계에 집중 | API 통합, 수익 공유, 공동 마케팅 | 파트너 채널 전환율, 파트너 수 |
```

### 첫 100명 사용자 획득 계획

```
타겟 사용자 프로필: [Persona에서]
어디서 찾을 것인가: [그들이 모이는 온/오프라인 장소]
어떤 훅으로 끌 것인가: [PR-FAQ의 Aha Moment에서]
어떻게 체험하게 할 것인가: [진입 장벽을 낮추는 방법]
초기 목표: [Y일 내에 X명이 제품을 체험]
```

### 출시 전략

| 전략 | 설명 | 적합한 경우 |
|------|------|-----------|
| 클로즈드 베타 | 10-50명을 수동 초대, 심층 피드백 수집 | PMF Level 1, 제품이 아직 다듬어지는 중 |
| 오픈 베타 | 공개하되 베타 라벨, 기대치 관리 | PMF Level 1-2, 더 많은 데이터 필요 |
| 소프트 런치 | 특정 시장이나 채널에서 저프로필 런치 | 전면 노출 없이 실제 환경 테스트 필요 |
| 정식 런치 | 전 채널 공식 출시 | PMF Level 2-3, 제품이 안정적 |

## 4.8 비즈니스 모델 & 프라이싱

**적용: 풀 모드 / 높은 완성도 / 독자가 경영진/영업인 경우**

### 수익 모델 선택

```
| 모델 | 작동 방식 | 적합한 경우 | 핵심 지표 |
|------|----------|-----------|----------|
| Freemium | 무료 기본 + 유료 프리미엄 | 대규모 사용자 베이스, 무료 티어가 가치를 보여줌 | 무료→유료 전환율 (목표 2-5%) |
| 구독 | 월간/연간 구독 | 지속적 가치를 제공하는 도구 또는 서비스 | MRR/ARR, 이탈률 |
| 사용량 기반 | 사용한 만큼 결제 | API, 클라우드 서비스, 거래 플랫폼 | 사용자당 평균 지출, 사용량 성장률 |
| 일회성 구매 | 일괄 구매 | 도구 소프트웨어, 강좌 | 평균 주문 가치, 재구매율 |
| 거래 수수료 | 거래당 일정 비율 | 플랫폼, 마켓플레이스 | GMV, 수수료율 |
| 사용자당 과금 | 사용자 수별 과금 | B2B 협업 도구 | 계정당 평균 좌석 수, 확장률 |
```

### 가치 기반 가격 정렬

```
핵심 가격 질문:
1. 제품이 사용자에게 얼마나 절약하거나 벌어주는가? (가치 앵커)
2. 사용자가 현재 대안에 얼마를 지불하고 있는가? (경쟁 기준)
3. 가격이 사용자가 받는 가치에 비례하는가? (공정성 인식)

가격 전략:
  - 모델 선택: [Freemium / 구독 / 사용량 기반 / ...]
  - 가격 범위: [가치 앵커와 경쟁 기준 기반]
  - 무료 티어 포함: [Aha Moment에 도달할 수 있되, 고급 니즈에는 충분하지 않게]
  - 유료 전환 트리거: [어떤 사용 행동이나 사용량이 업그레이드를 트리거하는가?]

⚠️ 흔한 가격 실수:
  - 가격이 너무 낮음 ("가치가 크지 않다"는 신호)
  - 무료 티어가 너무 관대 (업그레이드 동기 없음)
  - 가격-가치 불일치 (기능 기반 vs. 가치 기반 가격 책정)
  - B2B에서 연간 계약 할인 없음 (현금 흐름 안정성 놓침)
```
