# 사전 단계: 기회 점검

> **참고: 각 모드에서 실행할 단계는 SKILL.md의 "모드 단계 시퀀스"가 권위 있는 기준입니다. 아래 내용은 참고용입니다.**

**적용: 풀 모드, 높은 완성도, 독자가 경영진/리더십인 경우**
**생략 가능: 퀵 모드, 빌드 직행 모드, 독자가 엔지니어/디자이너인 경우**

사용자가 0-to-1 제품을 처음부터 구축하는 경우, 먼저 다음 5개 질문을 거쳐가세요. 어떤 질문에서든 "아니오"가 나오면 재검토가 필요한 신호입니다:

```
| # | 평가 질문 | 사용자 답변 | 평가 |
|---|----------|-----------|------|
| 1 | 이것은 실제로 긴급한 사용자 페인포인트를 해결하나요? 최초 수혜 고객은 누구인가요? 어떻게 찾을 것인가요? | | ✅/⚠️ |
| 2 | 이 문제를 해결하는 데 고유한 장점이 있나요? 타겟 고객이 최소 주 1회 이상 사용할 것인가요? 시장이 충분히 큰가요? | | ✅/⚠️ |
| 3 | 현재 리소스로 2-3년 내에 사용 가능한 제품을 구축할 수 있나요? | | ✅/⚠️ |
| 4 | 경쟁 구도는 어떤가요? 이길 수 있나요? 차별화 포인트는 무엇인가요? | | ✅/⚠️ |
| 5 | 지속 가능한 사용자 성장과 수익화 경로가 있나요? | | ✅/⚠️ |
```

## DHM 빠른 점검 (Gibson Biddle / Netflix)

이 기회가 달성할 수 있는지 확인하세요:
- **D (Delight)**: 사용자의 기대를 놀라게 하고 초과할 수 있는가?
- **H (Hard to copy)**: 경쟁자가 복제하기 어려운가?
- **M (Margin-enhancing)**: 규모가 커질수록 마진이 개선되는가?

일부 시장 신호가 불명확하다면 방향성 신호를 찾으세요:
- 거시 트렌드 (예: AI 도입으로 인한 워크플로우 대체 기회)
- 행동 변화 (사용자가 이미 우회 방법을 사용 중이라면 실제 수요가 있다는 의미)
- 유사 시장 (검증된 비교 가능한 시나리오 찾기)

## 팀 열정 점검

팀이 이 문제 영역에 진정한 열정을 가지고 있는지 확인하세요. 내재적 동기가 부족한 팀은 PMF로 가는 길에서 필연적으로 흔들립니다.
