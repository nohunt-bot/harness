# 多格式文件匯出規則

> 使用者觸發 `/export [format]` 或在流程結束後選擇匯出格式時載入。
> 首次使用時先載入 `rules-document-tools.md` 確認工具已安裝。

## 支援的匯出格式

| 格式 | 指令 | 轉換路徑 | 工具 |
|------|------|---------|------|
| PDF | `/export pdf` | HTML+CSS → Playwright → pikepdf 書籤 | Playwright MCP + pikepdf |
| DOCX | `/export docx` | MD → Pandoc | pandoc + reference.docx |
| PPTX | `/export pptx` | MD → Pandoc | pandoc |
| HTML | `/export html` | 已有 `06-html-report.md` 規則 | 內建 |
| Markdown | `/export md` | 直接輸出 | 內建 |

## PDF 匯出（核心流程）

### 路徑判斷

PDF 匯出有兩條路徑，根據來源內容自動選擇：

| 來源 | 路徑 | 原因 |
|------|------|------|
| 規劃成果直接匯出（PRD / 增量更新 / 規格書） | **路徑 A：setContent** | 無外部 JS 依賴，可內嵌所有資源 |
| HTML 企劃報告轉 PDF（含 Mermaid 圖表 / CDN 字體） | **路徑 B：HTTP Server** | 需要載入外部 JS (mermaid.js) 和 CDN 字體 |

### 路徑 A：獨立文件 → PDF（setContent）

適用於：PRD、功能擴充增量更新、規格書、任何不含外部 JS 的內容。

**步驟 1：生成 HTML 內容**

Claude 讀取 `references/templates/prd-style.css`，生成完整的 HTML 文件：

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
<meta charset="UTF-8">
<style>
/* 讀取 prd-style.css 的完整內容，內嵌於此 */
</style>
</head>
<body>

<!-- 封面頁 -->
<div class="cover-page">
  <h1>[文件標題]</h1>
  <div class="subtitle">[副標題或一句話描述]</div>
  <div class="version-badge">[版本號]</div>
  <div class="meta-info">
    <strong>PM</strong> [PM 名稱]<br>
    <strong>日期</strong> [ISO 日期]<br>
    <strong>狀態</strong> [狀態]
  </div>
</div>

<!-- 圖例（僅增量更新文件需要） -->
<div class="diff-legend">
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-new"></div> 本次新增</div>
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-upd"></div> 本次修改</div>
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-unchanged"></div> 不變</div>
</div>

<!-- 目錄頁 -->
<div class="toc-page">
  <h2>目錄</h2>
  <ul class="toc">
    <!-- Claude 根據文件的 h2 標題自動生成目錄項目 -->
    <li>
      <span class="toc-title">[章節名稱]</span>
      <span class="toc-dots"></span>
      <span class="toc-page-num">[頁碼]</span>
    </li>
  </ul>
</div>

<!-- 正文內容 -->
[正文 HTML]

<!-- 頁尾 -->
<div class="doc-footer">
  [文件編碼] [版本] ｜ [日期] ｜ Generated by Product Playbook
</div>

</body>
</html>
```

**重要**：
- CSS 必須完全內嵌（不使用外部連結）
- 字體使用系統字體：`"PingFang TC", "Noto Sans TC", system-ui`（Playwright 會自動嵌入 PDF）
- 不包含任何 `<script>` 標籤或外部資源引用
- 目錄頁碼為預估值（生成 PDF 後可能需微調）

**步驟 2：Playwright 渲染 PDF**

將 HTML 內容儲存到 `/tmp/export-{timestamp}.html`，然後呼叫 Playwright MCP：

```javascript
// mcp__plugin_playwright_playwright__browser_run_code
async (page) => {
  const fs = require('fs');
  const html = fs.readFileSync('/tmp/export-{timestamp}.html', 'utf8');
  await page.setContent(html, { waitUntil: 'networkidle' });
  await page.pdf({
    path: '{output_path}',
    format: 'A4',
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' },
    printBackground: true,
    displayHeaderFooter: true,
    headerTemplate: '<div></div>',
    footerTemplate: '<div style="font-size:8pt;color:#999;text-align:center;width:100%;">p.<span class="pageNumber"></span> / <span class="totalPages"></span></div>'
  });
  return 'PDF generated';
}
```

> 注意：若 HTML 內容過大（>500KB），setContent 可能超時。此時改用路徑 B（HTTP Server）。

**步驟 3：pikepdf 注入書籤**

```python
import pikepdf, re

pdf = pikepdf.open('{output_path}')

# 從 HTML 中提取 h2 標題作為書籤
# Claude 在生成 HTML 時需記錄每個 h2 對應的預估頁碼
bookmarks = [
    # (標題, 頁碼索引 0-based)
    ("修訂紀錄", 2),
    ("User Story", 4),
    ("開發架構說明", 6),
    # ... 依實際內容生成
]

with pdf.open_outline() as outline:
    for title, page_idx in bookmarks:
        page_idx = min(page_idx, len(pdf.pages) - 1)
        outline.root.append(
            pikepdf.OutlineItem(title, page_idx)
        )

# 設定 PDF 元資料
with pdf.open_metadata() as meta:
    meta['dc:title'] = '[文件標題]'
    meta['dc:creator'] = ['[PM 名稱]']
    meta['pdf:Producer'] = 'Product Playbook + Playwright'

pdf.save('{output_path}')
```

**步驟 4：通知使用者**

```
✅ PDF 已匯出至：{output_path}
  📄 {頁數} 頁 ｜ {檔案大小}
  📑 含書籤導航（{書籤數} 個章節）
```

### 路徑 B：HTML 報告 → PDF（HTTP Server）

適用於：含 Mermaid 圖表、CDN 字體、互動元素的 HTML 報告。

**步驟 1：HTML 報告已生成**

HTML 報告由 `06-html-report.md` 規則生成，已存為 `/tmp/report-{timestamp}.html`。

**步驟 2：啟動 HTTP Server**

```bash
cd /tmp && python3 -m http.server 18899 &
echo $!  # 記錄 PID
```

**步驟 3：Playwright 渲染**

```javascript
// mcp__plugin_playwright_playwright__browser_run_code
async (page) => {
  await page.goto('http://localhost:18899/report-{timestamp}.html', {
    waitUntil: 'networkidle',
    timeout: 30000
  });
  // 等待 Mermaid 圖表渲染完成
  await page.waitForTimeout(3000);
  // 展開所有手風琴（<details>）元素
  await page.evaluate(() => {
    document.querySelectorAll('details').forEach(d => d.open = true);
  });
  await page.pdf({
    path: '{output_path}',
    format: 'A4',
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' },
    printBackground: true,
    displayHeaderFooter: true,
    headerTemplate: '<div></div>',
    footerTemplate: '<div style="font-size:8pt;color:#999;text-align:center;width:100%;">p.<span class="pageNumber"></span> / <span class="totalPages"></span></div>'
  });
  return 'PDF generated';
}
```

**步驟 4：關閉 HTTP Server + pikepdf 書籤**

```bash
kill {SERVER_PID}
```

然後執行與路徑 A 相同的 pikepdf 書籤注入步驟。

### Playwright MCP 不可用時的 Fallback

若 Playwright MCP 未連線，使用 Node.js 直接執行：

```bash
node -e "
const { chromium } = require('playwright');
const fs = require('fs');
(async () => {
  const browser = await chromium.launch({ headless: true });
  const page = await browser.newPage();
  const html = fs.readFileSync('/tmp/export-{timestamp}.html', 'utf8');
  await page.setContent(html, { waitUntil: 'networkidle' });
  await page.pdf({
    path: '${OUTPUT_PATH}',
    format: 'A4',
    printBackground: true,
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' }
  });
  await browser.close();
  console.log('PDF generated');
})();
"
```

## DOCX 匯出

### 步驟 1：生成 Markdown

Claude 將規劃成果整理為乾淨的 Markdown 格式，存為 `/tmp/export-{timestamp}.md`。

### 步驟 2：Pandoc 轉換

```bash
pandoc /tmp/export-{timestamp}.md \
  -o "{output_path}" \
  --from markdown \
  --to docx \
  --reference-doc="references/templates/reference.docx" \
  --toc \
  --toc-depth=2
```

若無 reference.docx 模板，省略 `--reference-doc` 參數（使用 Pandoc 預設樣式）。

### 步驟 3：通知使用者

```
✅ Word 文件已匯出至：{output_path}
  📄 含目錄 ｜ {檔案大小}
```

## PPTX 匯出

```bash
pandoc /tmp/export-{timestamp}.md \
  -o "{output_path}" \
  --from markdown \
  --to pptx \
  --slide-level=2
```

> 提示：Pandoc PPTX 品質有限，建議使用者將 PPTX 當作大綱，再用 PowerPoint/Keynote 美化。

## 增量更新文件的特殊處理

當匯出的是 Feature Extension 或 Revision 模式的增量更新文件時：

### Diff 視覺標記規則

1. **`[NEW]` 標記的段落**：
   - 使用 `<span class="new">NEW</span>` 標記
   - 所在表格行加 `class="new-row"`
   - 段落加 `class="diff-added"`

2. **`[UPDATED]` 標記的段落**：
   - 使用 `<span class="upd">UPDATED</span>` 標記
   - 所在表格行加 `class="upd-row"`
   - 若有原始內容，原始內容加 `class="diff-removed"`，新內容加 `class="diff-added"`

3. **未變更的段落**：正常顯示，不加任何特殊標記

4. **文件開頭加圖例**：
```html
<div class="diff-legend">
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-new"></div> 本次新增
  </div>
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-upd"></div> 本次修改
  </div>
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-unchanged"></div> 無標記 = 不變
  </div>
</div>
```

## 輸出路徑規則

預設輸出到 `~/Downloads/`，檔名格式：

```
[文件編碼] [文件標題] [版本].{ext}

範例：
[D-221] 凱基個人虛擬帳號 v1.0.14 完整合併版.pdf
Product Planning Report - UPASS Virtual Account.docx
```

若使用者指定路徑，使用使用者指定的路徑。

## 品質檢查清單

匯出前，Claude 自我檢查：

- [ ] HTML 中無殘留的 Markdown 語法（`**`, `##`, `|---|`）
- [ ] 所有表格的行數和列數與原始內容一致
- [ ] `[NEW]` / `[UPDATED]` 標記正確套用對應 CSS class
- [ ] 封面頁資訊（標題、版本、PM、日期）正確
- [ ] 目錄項目與正文 h2 標題一致
- [ ] CSS 完全內嵌（路徑 A）或 CDN 可存取（路徑 B）
- [ ] 書籤數量與 h2 標題數量一致
