# 阶段四：Deliver — 假设验证 + 产品规格摘要

## 4.5 假设验证计划

**适用于：中/高完整性 / 产出对象为数据科学家/自己**

```
| 核心假设 | 验证方法 | 验证时间点 | 成功标准 | 如果假设错误，我们将... |
|---------|---------|-----------|---------|----------------------|
| | Alpha 用户访谈 / 数据 | | | |
```

原则：质化验证（用户访谈、日记研究）+ 量化验证（数据）并重；对自己的确认偏误保持警惕。

## 4.6 产品规格摘要

产品规格摘要是所有模式的最终产出（快速模式产出一页式方向摘要，结构不同但精神一致）。
产品规格摘要分为三个区块，对应不同阅读深度：

### 区块一：决策摘要（30 秒能看完）

```
**产品一句话**：[PR-FAQ 标题]
**为谁**：[目标 Persona 一句话描述]
**核心价值**：[JTBD 功能性工作 + 情感性工作，一句话]
**北极星指标**：[指标名称 + 定义]
**PMF 目前等级**：[Level 1-4 + 一句话说明]
```

### 区块二：执行边界（2 分钟能看完）

```
**核心 JTBD 陈述**：[Target Customer] + 想要在 [Job Context] + 完成 [Job]
**Aha Moment**：当用户完成 [行为]，他们体验到核心价值。目标在 [X] 内到达。
**MVP 必须有**：[3-5 个最关键的功能/能力]
**Not Doing（此版本刻意不做）**：[明确排除项 + 理由]
**关键假设**：[需要验证的 2-3 个核心假设]
**Pre-mortem 主要风险**：[最可能导致失败的 2-3 个原因]
**Hero Metric**：[1-2 个最重要的成功指标 + 目标值]
```

**商业模式**：[收费模式 + 价格策略，若已执行]
