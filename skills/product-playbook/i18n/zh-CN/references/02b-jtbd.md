# 阶段一：Discovery — JTBD 分析

## 1.3 JTBD（Jobs to Be Done）分析

> 「分析的单位不是消费者，而是消费者试图完成的那件工作。」— Clayton Christensen

**JTBD 描述公式：**
```
[瞄准的顾客] + 想要在 [什么工作情境 Job Context] + 完成 [什么工作 Jobs]
```

范例：正在比较房贷方案的首购族，想要在深夜不方便联络银行时，快速估算每月还款金额，以便向伴侣说明财务规划。

**JTBD 四种类型分析表：**

```
| JTBD 类型 | 定义 | Persona 1 | Persona 2 |
|-----------|------|-----------|-----------|
| 功能性工作（Functional Job） | 完成特定任务或达成功能性目标 | | |
| 情感性工作（Emotional Job） | 感受或希望有什么感受 | | |
| 社交性工作（Social Job） | 希望在他人眼中呈现什么形象 | | |
| 工作情境（Job Context） | 在什么情况下需要完成这份工作 | | |
```

**JTBD 深挖五问：**
1. **根本问题**：用户表达的需求背后，真正想解决的是什么？
2. **现行限制**：哪些解法因为某些条件限制被排除在外？
3. **现行做法**：用户现在用什么方法应对？做了哪些 workaround？
4. **差距**：现行做法哪里无法令他们满意？（这个差距就是你的机会）
5. **理想解法**：如果限制消除，他们理想中的解法是什么样子？

**Teresa Torres 用户访谈最佳实践：**
- 聚焦在用户过去的**实际行为**，而不是假设性的未来行为
- 问「上一次你遇到这个问题时，你怎么做的？」而不是「你希望有什么功能？」
- 最常见错误：问假设性问题、带入解法偏见、没有追问细节

### 📝 JTBD 品质自检清单

Claude 产出 JTBD 后必须自我检查（每项必须标记 ✅ 或 ❌，❌ 项目必须说明如何改善）：
- [ ] 是否包含具体情境？（不是「随时随地」，而是「在深夜不方便联络银行时」）
- [ ] 是否聚焦在一个核心工作？（不是三个 Job 合并成一句）
- [ ] 功能性/情感性/社交性工作是否都有识别？
- [ ] 是否能用来判断「这个解法有没有解决这个 Job」？
- [ ] 是否包含「现行做法」和「差距」？（差距 = 机会）
- [ ] 深挖五问的 Q5 是否触及情感动机/职业认同/心理恐惧？（非功能性描述）

**执行规则（Hard Gate）**：
- 必须逐项标记 ✅ 或 ❌，不允许保留空白 [ ] 或不说明的 ✅ 列表
- 若所有项目都是 ✅，必须额外说明「这份分析最弱的一个环节是什么，如何补强」
- ❌ 常见问题：太抽象、太多 Job 合并、缺少情境、用产品功能替代 Job 描述、Q5 停在功能层面

### 🏢 B2B 产品专用深度要求

**B2B 产品（含 B2B2C）必须完成以下分析：**

#### 组织层级 Job 分析（必填，至少覆盖 2 层）

| 层级 | 说明 | 范例 |
|------|------|------|
| **Strategic Job** | 组织/管理层的跨部门需求 | 合规稽查、成本控制、人力资源优化 |
| **Operational Job** | 流程/部门主管的协调需求 | 审批流程管理、跨部门资讯同步 |
| **Task Job** | 个别使用者的日常操作需求 | 填写表单、查询状态、汇出报表 |

#### 买方（Buyer）vs 使用者（User）分析（必填）

若买方与使用者不同人，必须分别分析其 JTBD：
- **买方 Job**：影响采购决策的工作（ROI 说明、风险降低、合规要求）
- **使用者 Job**：日常操作时需要完成的工作（效率提升、错误减少）
- 若同一人，说明「为何此场景中决策者即使用者」

#### 深挖五问 B2B 强化版

深挖五问的 **Q5 必须到达以下任一层面**（举例）：
- ✅ 职业认同：「她怕在高层面前出糗，因为这份报表代表她部门的专业度」
- ✅ 情感动机：「他想要在下属面前展现自己掌握数字的能力」
- ✅ 心理恐惧：「她最怕的是被稽核单位抓到流程漏洞，因为上次已经被警告过一次」
- ❌ 不合格范例：「他需要一个更好的工具来提高效率」（停留在功能层面）

#### 竞品替代品分析（必填）

列出目前使用者实际使用的替代方案：
- 至少 2 个具名现行工具（如 Line/Excel/纸本/Email/口头沟通）
- 每个工具必须说明其「根本缺陷」：不是功能差，而是「为什么这个缺陷被接受却没被解决」（组织惯性？切换成本？上级不在意？）

### 📋 用户访谈计划模板

```
## 用户访谈计划

**研究目标**：了解 [目标 Persona] 在 [Job Context] 中的 [具体问题]
**受访者筛选条件**：
  - 必须在过去 [X 天/周] 内有过 [具体行为]
  - 排除：[哪些人不适合，例如内部员工、已知的忠实用户]

**核心问题（5-7 题）**：
1. 上一次你遇到 [问题] 时，可以带我走一遍你是怎么处理的吗？（行为回溯）
2. 在那个过程中，最让你困扰/花最多时间的是哪个环节？（痛点定位）
3. 你有试过其他方法吗？为什么用/不用？（现行替代方案）
4. 如果那个环节可以变得更好，「好」对你来说是什么样子？（理想状态）
5. 这件事多久会发生一次？最近一次是什么时候？（频率和急迫性）
6. 除了你以外，还有谁会受到这个问题的影响？（利害关系人）
7. 如果要用 1-10 分评量这个问题的严重程度，你会打几分？为什么？（量化痛感）

**追问策略**：
  - 当受访者说「通常我会...」→ 追问「上一次具体是什么情况？」
  - 当受访者提到情绪 → 追问「那个感觉具体是什么？」
  - 当受访者提到某个工具/方法 → 追问「是什么让你选择这个方式？」

**记录格式**：
  - 逐字稿 or 录音
  - 访后 24 小时内标记：关键引言 / 痛点 / 惊喜发现 / 与假设不符之处
```
