# ステージ4：Deliver — PMF + GTM + ビジネスモデル

## 4.3 Todd Jackson（First Round Capital）の4段階PMFフレームワーク

```
| レベル | 名称 | 特徴 | あなたのタスク | 典型的な企業ステージ |
|-------|------|----------------|-----------|----------------------|
| レベル1 | 初期PMF | ソリューションに対して支払い意思のある顧客が3-5社見つかった | これらの初期ユーザーに深く対応し、再現可能なパターンを見つける | プレシード / シード |
| レベル2 | 発展中のPMF | 口コミが始まるが、成長はまだ不安定 | スケーラブルな獲得チャネルを見つけ、リテンションを改善 | シード / シリーズA |
| レベル3 | 強力なPMF | 「魚が船に飛び込んでくる」— 紹介が主要な成長ドライバーに | スケーラブルな需要チャネルを構築、効率メトリクスを改善 | シリーズA / B |
| レベル4 | 極限のPMF | メトリクスが堅牢、企業は100人超、TAM拡大を検討 | 新市場への参入または新プロダクトのリリース | シリーズC以降 |
```

**Four P's（レベル1-2でPMFを見つけるための重要質問）：**
1. Problem：解決している問題は本当に十分に緊急で重要か？
2. Promise：プロダクトはユーザーにどんな約束をしているか？ユーザーはそれを信じるか？
3. Product：プロダクトは本当にその約束を果たしているか？
4. Pitch：ターゲットユーザーにどうやってプロダクトを発見させ、信じさせるか？

## 4.7 GTM戦略（Go-to-Market）

**対象：フルモード / 高完成度 / オーディエンスが経営層/セールス/マーケティングの場合**

プロダクトが完成したら、どうやってターゲットユーザーに知ってもらい、使い始めてもらうか？

### 獲得チャネルの選択

```
| チャネルタイプ | 最適な用途 | 典型的な手法 | 主要メトリクス |
|-------------|----------|----------------|-------------|
| Product-Led Growth（PLG） | ユーザーがセルフサーブで価値を体験できるプロダクト | 無料トライアル、Freemium、バイラル招待 | サインアップ転換率、アクティベーション率、招待率 |
| コンテンツ＆コミュニティ | ターゲットユーザーが共通の情報ニーズやコミュニティを持つ | ブログ、SEO、コミュニティ構築、KOL | トラフィック、コンテンツ転換率、コミュニティエンゲージメント |
| 有料獲得 | 高速検証またはROIが明確な場合 | SEM、ソーシャル広告、リターゲティング | CAC、ROAS、LTV/CAC |
| Sales-Led | B2B高ACV、デモと交渉が必要 | SDR、AE、デモ、エンタープライズ契約 | パイプライン、成約率、ACV |
| パートナーシップ | ターゲットユーザーが特定のプラットフォームやエコシステムに集中 | API統合、レベニューシェア、共同マーケティング | パートナーチャネル転換率、パートナー数 |
```

### 最初の100ユーザー獲得プラン

```
ターゲットユーザーのプロフィール：[ペルソナから]
彼らがいる場所：[オンライン/オフラインで集まる場所]
引きつけるフック：[PR-FAQのAha Momentから]
試してもらう方法：[障壁を下げる方法]
初期目標：[Y日以内にXユーザーにプロダクトを試してもらう]
```

### ローンチ戦略

| 戦略 | 説明 | 最適な用途 |
|----------|------------|----------|
| クローズドベータ | 10-50人を手動で招待、深いフィードバックを収集 | PMFレベル1、プロダクトがまだ洗練中 |
| オープンベータ | 公開だがBetaラベル付き、期待値を管理 | PMFレベル1-2、より多くのデータが必要 |
| ソフトローンチ | 特定の市場やチャネルで控えめにリリース | フル露出なしで実環境テストが必要 |
| フルローンチ | 全チャネル正式リリース | PMFレベル2-3、プロダクトが安定 |

## 4.8 ビジネスモデル＆プライシング

**対象：フルモード / 高完成度 / オーディエンスが経営層/セールスの場合**

### 収益モデルの選択

```
| モデル | 仕組み | 最適な用途 | 主要メトリクス |
|-------|-------------|----------|-------------|
| Freemium | 無料基本プラン + 有料プレミアム | 大規模ユーザーベースのプロダクト、無料プランで価値を実証 | 無料→有料転換率（目標2-5%） |
| サブスクリプション | 月額/年額課金 | 継続的な価値を提供するツールやサービス | MRR/ARR、チャーン率 |
| 従量課金 | 使った分だけ支払い | API、クラウドサービス、トランザクションプラットフォーム | ユーザーあたり平均支出、利用成長率 |
| 買い切り | 一括購入 | ツールソフトウェア、コース | 平均注文額、リピート率 |
| トランザクションフィー | 取引ごとに手数料 | プラットフォーム、マーケットプレイス | GMV、テイク率 |
| シートベース | ユーザーごとに課金 | B2Bコラボレーションツール | アカウントあたり平均シート数、拡張率 |
```

### 価値ベースのプライシングアラインメント

```
コアプライシング質問：
1. プロダクトはユーザーにどれだけ節約させる、またはどれだけ稼がせるか？（価値アンカー）
2. ユーザーは現在、代替品にいくら支払っているか？（競合参照）
3. 価格はユーザーが受け取る価値に見合っているか？（公正性の認識）

プライシング戦略：
  - モデル選択：[Freemium / サブスクリプション / 従量課金 / ...]
  - 価格帯：[価値アンカーと競合参照に基づく]
  - 無料プランに含まれるもの：[Aha Momentに到達するのに十分だが、高度なニーズには不十分]
  - 有料トリガーポイント：[どんな利用行動や量がアップグレードをトリガーするか？]

⚠️ よくあるプライシングの間違い：
  - 価格が低すぎ（「たいした価値がない」というシグナル）
  - 無料プランが寛大すぎ（アップグレード動機がない）
  - 価格と価値のミスアラインメント（機能ベースプライシング vs 価値ベースプライシング）
  - B2Bで年間契約割引なし（キャッシュフロー安定性を逃す）
```
