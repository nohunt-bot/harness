# 🏢 B2B / B2C差異調整

> プロダクトタイプ確認後に読み込みます。Claudeは各フレームワーク内で以下の調整を自動適用します。

| フレームワーク | B2C調整 | B2B調整 |
|-----------|----------------|----------------|
| **ペルソナ** | 個人のモチベーション/習慣でセグメント | 「購買意思決定者」と「実際のエンドユーザー」を2つのペルソナタイプとして区別；「組織規模」と「調達プロセス」フィールドを追加 |
| **JTBD** | 個人の感情的・社会的ジョブにフォーカス | 「組織レベルのジョブ」を追加：会社のどんな目標達成に貢献するか？組織内で個人はどんなジョブを達成するか？ |
| **PMF評価** | 主にDAU/MAU、リテンション率、Sean Ellis Scoreに基づく | 主に「有料顧客 + 拡張率 + NPS」に基づく；レベル1 = 有料顧客3-5社 |
| **Aha Moment** | 初回使用時にコア価値を体験 | オンボーディング/トライアル期間まで出現しない可能性；「Time-to-Value」メトリクスを設計 |
| **MVP** | 完全な軽量プロダクトが可能 | Pilot（特定顧客向けのカスタマイズデプロイ）が必要な場合あり |
| **GTM** | 主にProduct-Led Growth（PLG）またはコミュニティ駆動 | 主にセールス駆動またはパートナーチャネル駆動 |
| **プライシング** | Freemium / サブスクリプション / 従量課金 | 年間契約 / シートベース / 従量課金 / カスタム見積 |
| **North Star** | ユーザーアクティビティまたはコアアクション完了数 | ARR、Net Revenue Retention、アクティブアカウント |
