# 🔧 フレームワークメニュー + 補助コマンド

> ユーザーが「フレームワーク一覧」「利用可能なフレームワークを見せて」とリクエストした時、または補助コマンドを使用した時に読み込みます。

## フレームワークの指定

**2つのトリガー方法：**

**方法A（ユーザーが直接フレームワーク名を指定）：** そのフレームワークのガイドフローに直接進む — 再確認不要。

**方法B（ユーザーが「フレームワークを選びたい」「全フレームワーク一覧」など）：** 以下のメニューを提示：

```
📚 利用可能なフレームワーク — 番号または名前を入力：

【ユーザー理解】
 1. JTBD（Jobs to Be Done）— ユーザーが本当に達成したいジョブを特定
 2. ペルソナ — 利用目的/タスク/モチベーション駆動のユーザープロフィール構築
 3. ユーザージャーニーマップ — 完全なユーザー体験ジャーニーをマッピング
 4. Continuous Discovery — ユーザーとの週次対話の習慣を構築

【問題定義】
 5. OST / Opportunity Solution Tree — 機会とソリューションを体系的にリンク
 6. ポジショニング / April Dunford — 真の競争領域と差別化を発見
 7. HMW — ペインポイントをデザイン質問にリフレーミング

【ソリューション設計】
 8. Working Backwards / PR-FAQ — ユーザーアウトカムから逆算してソリューションを設計
 9. Pre-mortem — 失敗が起きる前に予測し防止
10. GEM — Growth / Engagement / Monetization 3次元優先順位付け
11. RICE — 定量的な機能優先順位付け
12. MVP — 最小実行可能プロダクトのスコープ定義

【戦略レイヤー】
13. Strategy / Strategy Blocks — Mission → Vision → Strategy の階層
14. DHMモデル — Delight / Hard to copy / Margin-enhancing の機会評価
15. LNOフレームワーク — Leverage / Neutral / Overhead の時間配分
16. Empowered Teams — エンパワードチーム vs フィーチャーチーム

【測定レイヤー】
17. North Star / North Star Metric — コアユーザー価値を表す単一メトリクスを定義
18. PMF — 4段階のProduct-Market Fit評価
19. Sean Ellis Score — PMF熱狂度の定量化

【ビジネスレイヤー】
20. ビジネスモデル＆プライシング — 収益モデル選択と価値ベースプライシングアラインメント
21. GTM戦略 — Go-to-Marketローンチと顧客獲得戦略

【開発ハンドオフ】
22. 開発ハンドオフ — CLAUDE.md + TASKS.md + TICKETS.md を生成してClaude Codeで開発ハンドオフ

フレームワーク番号または名前を入力（複数選択可、カンマ区切り）：
```

## ディスカバリーをスキップ / ビルドに直行

ユーザーが「ユーザーリサーチをスキップ」「問題はもう分かっている」「Developに直行」と言った場合、`references/rules-build.md`を読み込み、ビルドモードのステップ順序に従ってください。

> ユーザーにリマインド：「ユーザーリサーチフェーズをスキップすることは、ソリューションが仮定に基づいて構築されることを意味します。実行後できるだけ早くContinuous Discoveryを実施して検証することをお勧めします。」

## 機能拡張モードのトリガー

- 「機能を追加」/「新しい機能を追加したい」/「既存プロダクトに新機能」→ 機能拡張モードをトリガー（`references/rules-build.md` → 機能拡張クイックパスを読み込み）

## 補助コマンド

| コマンド | 動作 |
|---------|----------|
| `「[フレームワーク]に切り替え」` | 即座に切り替え、完了済み内容を保持 |
| `「ターゲットオーディエンスを変更したい」` | フレームワーク優先度とプレゼンテーションスタイルを再調整 |
| `「このステップをスキップ」` | 必要性をリマインドした後、決定を尊重して次のステップへ |
| `「[ステップ/フレームワーク名]に戻る」` | 指定ステップに戻り再ガイド（`references/rules-change-propagation.md`参照） |
| `「簡潔に」` / `「詳しく」` | 要点に凝縮 / 詳細な分析を追加 |
| `「レポートを生成」` | `references/06-html-report.md`を読み込み、HTML企画レポートを作成 |
| `「PRDを生成」` / `「エンジニアドキュメントを生成」` | `references/04b-solutions.md`を読み込み、PR-FAQ + MVP + User Story + Pre-mortemを統合し、**自動的にフローチャート（Mermaid） + DBスキーマ（Mermaid ERD） + UIワイヤーフレーム（HTML）も生成** |
| `「フローチャートを生成」` / `「フローチャートを描いて」` | Mermaid構文でフローチャートを出力（単独トリガー） |
| `「DBスキーマを生成」` / `「データベースを設計」` | Mermaid ERD構文でDBスキーマを出力（単独トリガー） |
| `「UIワイヤーフレームを生成」` / `「ワイヤーフレームを描いて」` | HTML/SVGで低忠実度UIワイヤーフレームを出力（単独トリガー） |
| `「プレゼンを生成」` / `「PPTを作って」` | システムのpptxスキルを呼び出し |
| `「このドキュメントを[オーディエンス]向けにアレンジ」` | 指定オーディエンス向けにフレームワークハイライトと言語を再編成 |
| `「15分しかない」` | 最も重要な3つの意思決定質問またはアクションを提供 |
| `「完成度評価を実行」` | どの領域が堅実で、どの領域にリスクがあるか評価 |
| `「仮定を見つけて」` | すべての未検証コア仮定を特定 |
| `「Pre-mortemを実行」` | 任意のソリューションに対して即座にPre-mortemを実行 |
| `「異なるオーディエンス向けバージョンを生成」` | 複数オーディエンス向けのサマリーを自動作成 |
| `「このプロダクトのPMFレベルは？」` | PMFレベルを判定し、次のマイルストーンを説明 |
| `「ボトルネックを見つけて」` | Aha Moment達成の最大の障害を分析 |
| `「これはリビジョンで、新プロダクトではない」` | リビジョンモードに切り替え（`references/rules-revision.md`を読み込み） |
| `「上司を説得して承認してもらいたい」` | ボスモードに切り替え — ビジネス価値とリソースロジックを強調 |
| `「開発を始めて」` / `「開発ハンドオフパッケージを生成」` | `references/07a-handoff-core.md`を読み込み、技術スタックを確認、完全な開発ハンドオフパッケージを生成 |
| `「プロジェクトをセットアップ」` / `「Claude Codeに接続」` | 上記と同じ |
| `「一時停止」` / `「保存」` / `「他のことをする」` | `references/rules-progress.md`に従い進捗を保存 |
| `「続行」` / `「企画に戻る」` | `references/rules-progress.md`に従い再開 |
| `「進捗をクリア」` / `「最初からやり直す」` | 進捗ファイルを削除しゼロから開始 |
| `/export [format]` | 指定フォーマットでエクスポート。format = `pdf` / `docx` / `pptx` / `html` / `md`。`references/rules-export-document.md` を読み込み、初回使用時はまず `references/rules-document-tools.md` を読み込みツールを確認。 |
| `/parse [file]` | アップロードされたドキュメントを Markdown に解析。PDF / DOCX / PPTX / 画像に対応。`references/rules-import-document.md` を読み込み、初回使用時はまず `references/rules-document-tools.md` を読み込みツールを確認。 |

**コンテキスト対応コマンドヒント**：各ステップ完了後、現在の進捗に基づいて最も関連性の高い利用可能なコマンドを2-3個積極的に提案してください。
