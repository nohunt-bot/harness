# マルチフォーマットドキュメントエクスポートルール

> ユーザーが `/export [format]` をトリガー、またはフロー終了後にエクスポート形式を選択した時に読み込みます。
> 初回使用時は先に `rules-document-tools.md` を読み込んでツールがインストール済みか確認してください。

## サポートされるエクスポート形式

| 形式 | コマンド | 変換パス | ツール |
|------|------|---------|------|
| PDF | `/export pdf` | HTML+CSS → Playwright → pikepdf ブックマーク | Playwright MCP + pikepdf |
| DOCX | `/export docx` | MD → Pandoc | pandoc + reference.docx |
| PPTX | `/export pptx` | MD → Pandoc | pandoc |
| HTML | `/export html` | 既存の `06-html-report.md` ルール | 内蔵 |
| Markdown | `/export md` | 直接出力 | 内蔵 |

## PDF エクスポート（コアフロー）

### パス判断

PDF エクスポートには2つのパスがあり、ソースコンテンツに基づいて自動選択：

| ソース | パス | 理由 |
|------|------|------|
| 企画成果の直接エクスポート（PRD / インクリメンタル更新 / 仕様書） | **パス A：setContent** | 外部 JS 依存なし、全リソースをインライン化可能 |
| HTML 企画レポート → PDF（Mermaid 図表 / CDN フォント含む） | **パス B：HTTP Server** | 外部 JS (mermaid.js) と CDN フォントの読み込みが必要 |

### パス A：独立ドキュメント → PDF（setContent）

適用対象：PRD、機能拡張インクリメンタル更新、仕様書、外部 JS を含まないすべてのコンテンツ。

**ステップ 1：HTML コンテンツを生成**

Claude が `references/templates/prd-style.css` を読み取り、完全な HTML ドキュメントを生成：

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
<meta charset="UTF-8">
<style>
/* prd-style.css の完全な内容をここにインライン */
</style>
</head>
<body>

<!-- 表紙ページ -->
<div class="cover-page">
  <h1>[ドキュメントタイトル]</h1>
  <div class="subtitle">[サブタイトルまたは一文の説明]</div>
  <div class="version-badge">[バージョン番号]</div>
  <div class="meta-info">
    <strong>PM</strong> [PM 名]<br>
    <strong>日付</strong> [ISO 日付]<br>
    <strong>ステータス</strong> [ステータス]
  </div>
</div>

<!-- 凡例（インクリメンタル更新ドキュメントのみ必要） -->
<div class="diff-legend">
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-new"></div> 今回新規追加</div>
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-upd"></div> 今回変更</div>
  <div class="diff-legend-item"><div class="diff-legend-swatch swatch-unchanged"></div> 変更なし</div>
</div>

<!-- 目次ページ -->
<div class="toc-page">
  <h2>目次</h2>
  <ul class="toc">
    <!-- Claude がドキュメントの h2 見出しから目次項目を自動生成 -->
    <li>
      <span class="toc-title">[セクション名]</span>
      <span class="toc-dots"></span>
      <span class="toc-page-num">[ページ番号]</span>
    </li>
  </ul>
</div>

<!-- 本文コンテンツ -->
[本文 HTML]

<!-- フッター -->
<div class="doc-footer">
  [ドキュメントコード] [バージョン] ｜ [日付] ｜ Generated by Product Playbook
</div>

</body>
</html>
```

**重要**：
- CSS は完全にインライン化（外部リンク不使用）
- フォントはシステムフォントを使用：`"PingFang TC", "Noto Sans TC", system-ui`（Playwright が自動的に PDF に埋め込み）
- `<script>` タグや外部リソース参照を含めない
- 目次ページ番号は推定値（PDF 生成後に微調整が必要な場合あり）

**ステップ 2：Playwright で PDF レンダリング**

HTML コンテンツを `/tmp/export-{timestamp}.html` に保存し、Playwright MCP を呼び出し：

```javascript
// mcp__plugin_playwright_playwright__browser_run_code
async (page) => {
  const fs = require('fs');
  const html = fs.readFileSync('/tmp/export-{timestamp}.html', 'utf8');
  await page.setContent(html, { waitUntil: 'networkidle' });
  await page.pdf({
    path: '{output_path}',
    format: 'A4',
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' },
    printBackground: true,
    displayHeaderFooter: true,
    headerTemplate: '<div></div>',
    footerTemplate: '<div style="font-size:8pt;color:#999;text-align:center;width:100%;">p.<span class="pageNumber"></span> / <span class="totalPages"></span></div>'
  });
  return 'PDF generated';
}
```

> 注意：HTML コンテンツが大きすぎる場合（>500KB）、setContent がタイムアウトする可能性があります。その場合はパス B（HTTP Server）に切り替え。

**ステップ 3：pikepdf でブックマーク注入**

```python
import pikepdf, re

pdf = pikepdf.open('{output_path}')

# HTML から h2 見出しをブックマークとして抽出
# Claude は HTML 生成時に各 h2 の推定ページ番号を記録する必要がある
bookmarks = [
    # (タイトル, ページインデックス 0-based)
    ("修訂紀錄", 2),
    ("User Story", 4),
    ("開発アーキテクチャ説明", 6),
    # ... 実際のコンテンツに基づいて生成
]

with pdf.open_outline() as outline:
    for title, page_idx in bookmarks:
        page_idx = min(page_idx, len(pdf.pages) - 1)
        outline.root.append(
            pikepdf.OutlineItem(title, page_idx)
        )

# PDF メタデータを設定
with pdf.open_metadata() as meta:
    meta['dc:title'] = '[ドキュメントタイトル]'
    meta['dc:creator'] = ['[PM 名]']
    meta['pdf:Producer'] = 'Product Playbook + Playwright'

pdf.save('{output_path}')
```

**ステップ 4：ユーザーに通知**

```
✅ PDF をエクスポートしました：{output_path}
  📄 {ページ数} ページ ｜ {ファイルサイズ}
  📑 ブックマークナビゲーション付き（{ブックマーク数} セクション）
```

### パス B：HTML レポート → PDF（HTTP Server）

適用対象：Mermaid 図表、CDN フォント、インタラクティブ要素を含む HTML レポート。

**ステップ 1：HTML レポートは生成済み**

HTML レポートは `06-html-report.md` ルールによって生成され、`/tmp/report-{timestamp}.html` に保存済み。

**ステップ 2：HTTP Server を起動**

```bash
cd /tmp && python3 -m http.server 18899 &
echo $!  # PID を記録
```

**ステップ 3：Playwright でレンダリング**

```javascript
// mcp__plugin_playwright_playwright__browser_run_code
async (page) => {
  await page.goto('http://localhost:18899/report-{timestamp}.html', {
    waitUntil: 'networkidle',
    timeout: 30000
  });
  // Mermaid 図表のレンダリング完了を待機
  await page.waitForTimeout(3000);
  // すべてのアコーディオン（<details>）要素を展開
  await page.evaluate(() => {
    document.querySelectorAll('details').forEach(d => d.open = true);
  });
  await page.pdf({
    path: '{output_path}',
    format: 'A4',
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' },
    printBackground: true,
    displayHeaderFooter: true,
    headerTemplate: '<div></div>',
    footerTemplate: '<div style="font-size:8pt;color:#999;text-align:center;width:100%;">p.<span class="pageNumber"></span> / <span class="totalPages"></span></div>'
  });
  return 'PDF generated';
}
```

**ステップ 4：HTTP Server を停止 + pikepdf ブックマーク**

```bash
kill {SERVER_PID}
```

その後、パス A と同じ pikepdf ブックマーク注入ステップを実行。

### Playwright MCP が利用不可の場合の Fallback

Playwright MCP が接続されていない場合、Node.js で直接実行：

```bash
node -e "
const { chromium } = require('playwright');
const fs = require('fs');
(async () => {
  const browser = await chromium.launch({ headless: true });
  const page = await browser.newPage();
  const html = fs.readFileSync('/tmp/export-{timestamp}.html', 'utf8');
  await page.setContent(html, { waitUntil: 'networkidle' });
  await page.pdf({
    path: '${OUTPUT_PATH}',
    format: 'A4',
    printBackground: true,
    margin: { top: '1.8cm', bottom: '1.8cm', left: '2cm', right: '2cm' }
  });
  await browser.close();
  console.log('PDF generated');
})();
"
```

## DOCX エクスポート

### ステップ 1：Markdown を生成

Claude が企画成果をクリーンな Markdown 形式に整理し、`/tmp/export-{timestamp}.md` に保存。

### ステップ 2：Pandoc で変換

```bash
pandoc /tmp/export-{timestamp}.md \
  -o "{output_path}" \
  --from markdown \
  --to docx \
  --reference-doc="references/templates/reference.docx" \
  --toc \
  --toc-depth=2
```

reference.docx テンプレートがない場合、`--reference-doc` パラメータを省略（Pandoc デフォルトスタイルを使用）。

### ステップ 3：ユーザーに通知

```
✅ Word ドキュメントをエクスポートしました：{output_path}
  📄 目次付き ｜ {ファイルサイズ}
```

## PPTX エクスポート

```bash
pandoc /tmp/export-{timestamp}.md \
  -o "{output_path}" \
  --from markdown \
  --to pptx \
  --slide-level=2
```

> ヒント：Pandoc の PPTX 品質には限界があるため、PPTX をアウトラインとして使用し、PowerPoint/Keynote で仕上げることを推奨します。

## インクリメンタル更新ドキュメントの特別処理

エクスポートするのが Feature Extension または Revision モードのインクリメンタル更新ドキュメントの場合：

### Diff ビジュアルマーキングルール

1. **`[NEW]` マークの段落**：
   - `<span class="new">NEW</span>` マークを使用
   - 所属するテーブル行に `class="new-row"` を追加
   - 段落に `class="diff-added"` を追加

2. **`[UPDATED]` マークの段落**：
   - `<span class="upd">UPDATED</span>` マークを使用
   - 所属するテーブル行に `class="upd-row"` を追加
   - 元のコンテンツがある場合、元のコンテンツに `class="diff-removed"`、新しいコンテンツに `class="diff-added"` を追加

3. **変更なしの段落**：通常表示、特別なマークなし

4. **ドキュメント冒頭に凡例を追加**：
```html
<div class="diff-legend">
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-new"></div> 今回新規追加
  </div>
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-upd"></div> 今回変更
  </div>
  <div class="diff-legend-item">
    <div class="diff-legend-swatch swatch-unchanged"></div> マークなし = 変更なし
  </div>
</div>
```

## 出力パスルール

デフォルトで `~/Downloads/` に出力、ファイル名形式：

```
[ドキュメントコード] [ドキュメントタイトル] [バージョン].{ext}

例：
[D-221] 凱基個人虛擬帳號 v1.0.14 完整合併版.pdf
Product Planning Report - UPASS Virtual Account.docx
```

ユーザーがパスを指定した場合、そのパスを使用。

## 品質チェックリスト

エクスポート前に Claude が自己チェック：

- [ ] HTML に残留 Markdown 構文がないか（`**`, `##`, `|---|`）
- [ ] すべての表の行数と列数が元のコンテンツと一致
- [ ] `[NEW]` / `[UPDATED]` マークに対応する CSS class が正しく適用されている
- [ ] 表紙ページ情報（タイトル、バージョン、PM、日付）が正確
- [ ] 目次項目と本文 h2 見出しが一致
- [ ] CSS が完全にインライン化（パス A）または CDN アクセス可能（パス B）
- [ ] ブックマーク数が h2 見出し数と一致
