# プレステップ：機会チェック

> **注意：各モードで実行するステップは、SKILL.mdの「モードステップ順序」で正式に定義されています。以下は参考情報です。**

**対象：フルモード、高完成度、オーディエンスが経営層/リーダーシップの場合**
**スキップ可能：クイックモード、ビルドモード、オーディエンスがエンジニア/デザイナーの場合**

ユーザーがゼロから0→1プロダクトを構築する場合、まず以下の5つの質問を確認してください。いずれかの質問で「いいえ」の場合、再検討のシグナルです：

```
| # | 評価質問 | ユーザーの回答 | 評価 |
|---|---------------------|---------------|------------|
| 1 | これは本当に緊急で切実なユーザーのペインポイントを解決しますか？最初に恩恵を受ける顧客は誰ですか？どうやって見つけますか？ | | ✅/⚠️ |
| 2 | この問題を解決する上で独自の優位性がありますか？ターゲット顧客は少なくとも週1回使いますか？市場は十分に大きいですか？ | | ✅/⚠️ |
| 3 | 現在のリソースで、2-3年以内に使えるプロダクトを構築できますか？ | | ✅/⚠️ |
| 4 | 競合状況はどうですか？勝てますか？差別化要因は何ですか？ | | ✅/⚠️ |
| 5 | ユーザー成長とマネタイズの持続可能な道筋がありますか？ | | ✅/⚠️ |
```

## DHMクイックチェック（Gibson Biddle / Netflix）

この機会は以下を実現できますか：
- **D（Delight）**：ユーザーの期待を上回り、驚かせることができるか？
- **H（Hard to copy）**：競合が模倣しにくいか？
- **M（Margin-enhancing）**：スケールに伴いマージンが改善するか？

市場シグナルが不明確な場合は、方向性を示すシグナルを探してください：
- マクロトレンド（例：AI普及によるワークフロー置換の機会）
- 行動変化（ユーザーが既にワークアラウンドを使用している＝実需の証拠）
- 類似市場（検証済みの類似シナリオを見つける）

## チームパッションチェック

チームがこの問題領域に対して本物の情熱を持っていることを確認してください。内発的なモチベーションが欠けているチームは、PMFへの道で必ず行き詰まります。
