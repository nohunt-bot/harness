# 1단계: 디스커버리 — OST + Journey Map

## 1.4 Opportunity Solution Tree (OST)

**적용: 풀 모드 / 높은 완성도**

OST는 제품 목표에서 시작하여 기회와 솔루션을 체계적으로 연결합니다:

```
[제품 목표 / 달성하려는 성과]
    │
    ├── [기회 1: 사용자 페인포인트 또는 니즈]
    │       ├── [솔루션 1a]
    │       └── [솔루션 1b]
    ├── [기회 2: 사용자 페인포인트 또는 니즈]
    │       └── [솔루션 2a]
    └── [기회 3: 사용자 페인포인트 또는 니즈]
            └── [솔루션 3a]
```

핵심 원칙:
- 목표(Outcome)는 기능이나 산출물이 아닌 측정 가능한 결과
- 기회는 사용자 리서치에서 나오지, 내부 브레인스토밍이 아님
- 솔루션은 기회에 매핑 — 기회를 건너뛰고 솔루션으로 바로 가지 마세요
- 넓게 먼저, 깊게 나중에: 모든 기회를 먼저 나열하고, 하나씩 솔루션 탐색

## 1.5 User Journey Map

**적용: 풀 모드 / 높은 완성도 / 독자가 디자이너인 경우**

**1단계: 개요 테이블**

```
**[Persona 이름] — 과업: [과업 설명]**

| 단계 | 핵심 행동 | 감정 | 주요 페인포인트 |
|------|----------|------|---------------|
| [단계 1] | [주요 행동 한 줄 설명] | [감정 + 이모지] | [가장 중요한 페인포인트] |
```

**2단계: 각 단계 상세 전개**

```
> **단계: [단계명]**
> - **행동(Doing)**: [이 단계에서 사용자가 실제로 하는 것]
> - **생각(Thinking)**: [사용자의 머릿속 생각, 가능하면 1인칭으로]
> - **감정(Feeling)**: [감정 상태와 이유]
> - **이해관계자(Stakeholder)**: [이 단계에 관여하는 사람]
> - **문제(Problem)**: [구체적인 어려움이나 불만]
```

**3단계: 그룹화**
- 단계가 너무 세분화된 경우, 더 큰 단계 그룹으로 병합
- 단계 간 페인포인트를 통합하고, 핵심 페인포인트를 표시

---

## 📎 이 단계의 파일 통합 팁

이 단계에서 사용자가 파일을 업로드하면, Claude는 다음 규칙에 따라 통합합니다:

| 업로드 내용 | 통합 대상 | 통합 작업 |
|------------|----------|----------|
| 화이트보드 / 손그림 플로우 다이어그램 | 1.5 Journey Map | 플로우를 인식하고 구조화된 테이블로 변환; 원래 감정 표시 유지 |
| 사용자 행동 데이터 (CSV) | 1.4 OST + 1.5 Journey Map | 데이터를 사용하여 가장 일반적인 행동 경로와 이탈률이 가장 높은 단계 검증 |
