# 1단계: 디스커버리 — JTBD 분석

## 1.3 JTBD (Jobs to Be Done) 분석

> "분석의 단위는 소비자가 아니라, 소비자가 완수하려는 과업(Job)입니다." — Clayton Christensen

**JTBD 선언문 공식:**
```
[타겟 고객] + 이 원하는 것은, [어떤 과업 맥락에서] + [어떤 과업]을 완수하는 것
```

예시: 모기지 옵션을 비교하는 첫 주택 구매자가, 은행에 연락할 수 없는 늦은 밤에 월 상환액을 빠르게 추산하여 배우자에게 재정 계획을 설명하고 싶어합니다.

**JTBD 4가지 유형 분석표:**

```
| JTBD 유형 | 정의 | Persona 1 | Persona 2 |
|-----------|------|-----------|-----------|
| 기능적 Job | 특정 과업을 완수하거나 기능적 목표를 달성 | | |
| 감정적 Job | 어떤 감정을 느끼거나 느끼고 싶은지 | | |
| 사회적 Job | 다른 사람들에게 어떻게 인식되고 싶은지 | | |
| Job 맥락 | 어떤 상황에서 이 과업을 완수해야 하는지 | | |
```

**JTBD 심층 5가지 질문:**
1. **근본 문제**: 사용자가 표현하는 니즈 뒤에, 실제로 해결하려는 것은 무엇인가?
2. **현재 제약**: 어떤 제약 때문에 어떤 솔루션이 배제되었는가?
3. **현재 우회 방법**: 사용자가 현재 어떻게 대처하고 있는가? 어떤 우회 방법을 만들었는가?
4. **갭**: 현재 우회 방법이 부족한 부분은 어디인가? (이 갭이 당신의 기회입니다)
5. **이상적 솔루션**: 제약이 없다면, 이상적인 솔루션은 어떤 모습일까?

**Teresa Torres 사용자 인터뷰 모범 사례:**
- 사용자의 **실제 과거 행동**에 집중하고, 가상의 미래 행동이 아닌
- "지난번 이 문제에 부딪혔을 때, 어떻게 하셨나요?"라고 질문하고, "어떤 기능을 원하세요?"가 아닌
- 가장 흔한 실수: 가상 질문, 솔루션 편향 도입, 세부사항 추적 부족

### 📝 JTBD 품질 체크리스트

Claude는 JTBD 산출물 작성 후 자체 점검해야 합니다 (각 항목은 ✅ 또는 ❌로 표시; ❌ 항목은 개선 방법 포함):
- [ ] 구체적인 맥락이 포함되어 있는가? ("언제 어디서나"가 아닌 — "은행에 연락할 수 없는 늦은 밤")
- [ ] 단일 핵심 Job에 집중하는가? (한 문장에 세 가지 Job이 섞여 있지 않은가)
- [ ] 기능적, 감정적, 사회적 Job이 모두 식별되었는가?
- [ ] "이 솔루션이 실제로 이 Job을 해결하는가?"를 평가하는 데 사용할 수 있는가?
- [ ] "현재 우회 방법"과 "갭"이 포함되어 있는가? (갭 = 기회)
- [ ] 심층 Q5가 감정적 동기 / 직업적 정체성 / 심리적 두려움에 도달하는가? (기능적 설명에만 머물지 않는가)

**실행 규칙 (하드 게이트):**
- 각 항목을 반드시 ✅ 또는 ❌로 표시 — 빈 [ ]이나 설명 없는 ✅ 목록은 허용되지 않음
- 모든 항목이 ✅이면, "이 분석의 가장 약한 부분은 무엇이고 어떻게 강화할 수 있는가"를 추가로 설명해야 함
- ❌ 흔한 문제: 너무 추상적, 여러 Job 혼합, 맥락 누락, 제품 기능으로 Job 설명 대체, Q5가 기능 수준에 머무름

### 🏢 B2B 제품 심층 분석 요구사항

**B2B 제품(B2B2C 포함)은 다음 분석을 완료해야 합니다:**

#### 조직 수준 Job 분석 (필수 — 최소 2개 수준 포함)

| 수준 | 설명 | 예시 |
|------|------|------|
| **전략적 Job** | 조직/경영 수준의 부서 간 니즈 | 규정 준수 감사, 비용 통제, 인력 최적화 |
| **운영적 Job** | 프로세스/부서 관리자 수준의 조율 니즈 | 결재 워크플로우 관리, 팀 간 정보 동기화 |
| **실무적 Job** | 개별 사용자의 일상 업무 니즈 | 양식 작성, 상태 확인, 보고서 내보내기 |

#### 구매자 vs. 사용자 분석 (필수)

구매자와 사용자가 다른 사람인 경우, JTBD를 별도로 분석하세요:
- **구매자 Job**: 구매 결정에 영향을 미치는 Job (ROI 정당화, 리스크 감소, 규정 준수 요구사항)
- **사용자 Job**: 일상 업무 중 완수해야 하는 Job (효율성 향상, 오류 감소)
- 동일 인물인 경우, "왜 이 시나리오에서 의사결정자가 사용자이기도 한지" 설명

#### 심층 5가지 질문 — B2B 강화 버전

**Q5는 다음 수준 중 최소 하나에 도달해야 합니다** (예시):
- ✅ 직업적 정체성: "그녀는 리더십 앞에서 무능해 보일까 봐 두려워합니다. 이 보고서가 그녀 부서의 신뢰성을 대표하기 때문입니다"
- ✅ 감정적 동기: "그는 직속 부하들에게 숫자를 확실히 파악하고 있다는 것을 보여주고 싶어합니다"
- ✅ 심리적 두려움: "가장 큰 두려움은 감사관이 프로세스 갭을 발견하는 것입니다 — 이미 한 번 경고를 받았기 때문입니다"
- ❌ 실패 예시: "그는 효율을 개선하기 위해 더 나은 도구가 필요합니다" (기능 수준에 머무름)

#### 경쟁 대안 분석 (필수)

사용자가 현재 실제로 사용하고 있는 대안을 나열하세요:
- 최소 2개의 구체적인 기존 도구 (예: Slack / Excel / 종이 양식 / 이메일 / 구두 소통)
- 각 도구에 대해 "근본적 결함"을 설명: 기능이 약한 것이 아닌, "왜 이 결함이 수용되고 미해결 상태인지" (조직적 관성? 전환 비용? 리더십이 관심 없음?)

### 📋 사용자 인터뷰 계획 템플릿

```
## 사용자 인터뷰 계획

**리서치 목표**: [타겟 Persona]가 [Job 맥락]에서 [특정 문제]를 어떻게 처리하는지 이해
**스크리닝 기준**:
  - 지난 [X일/주] 이내에 [특정 행동]을 경험한 사람이어야 함
  - 제외: [적합하지 않은 사람 — 예: 내부 직원, 알려진 파워 유저]

**핵심 질문 (5-7개)**:
1. 지난번 [문제]에 부딪혔을 때, 어떻게 처리하셨는지 과정을 알려주실 수 있나요? (행동 회상)
2. 그 과정에서 가장 답답하거나 시간이 많이 걸린 부분은 무엇이었나요? (페인포인트 식별)
3. 다른 방법을 시도해 보셨나요? 왜 혹은 왜 안 하셨나요? (현재 대안)
4. 그 부분이 더 나아진다면, "더 나은" 상태란 어떤 모습일까요? (이상적 상태)
5. 이런 일이 얼마나 자주 발생하나요? 마지막으로 언제였나요? (빈도와 긴급성)
6. 당신 외에 이 문제의 영향을 받는 사람이 또 있나요? (이해관계자 매핑)
7. 1-10점 척도로, 이 문제가 얼마나 심각한가요? 왜 그렇게 생각하시나요? (페인 정량화)

**후속 전략**:
  - 인터뷰 대상자가 "보통은..."이라고 말할 때 → "지난번에 구체적으로 어떤 일이 있었나요?" 질문
  - 인터뷰 대상자가 감정을 언급할 때 → "그 감정을 좀 더 구체적으로 설명해 주실 수 있나요?" 질문
  - 인터뷰 대상자가 도구/방법을 언급할 때 → "그 방법을 선택하게 된 이유가 무엇인가요?" 질문

**기록 형식**:
  - 축어 녹취 또는 녹음
  - 인터뷰 후 24시간 이내에 태그: 핵심 인용문 / 페인포인트 / 놀라운 발견 / 가정에 반하는 내용
```

## 📎 이 단계의 파일 통합 참고사항

이 단계에서 사용자가 파일을 업로드하면, Claude는 다음과 같이 통합합니다:

| 업로드 내용 | 통합 대상 | 통합 작업 |
|------------|----------|----------|
| 사용자 인터뷰 녹취록 / 녹음 텍스트 | 1.1 Persona + 1.3 JTBD | 추출: 사용자 배경 → Persona 항목; 페인포인트 + 현재 우회 방법 → JTBD 심층 5가지 질문; 감정적 반응 → 감정적 / 사회적 Job |
| 경쟁사 앱 스크린샷 | 1.3 JTBD (현재 우회 방법) | 사용자의 "현재 대안"으로 식별, 우회 방법과 갭 분석 |
