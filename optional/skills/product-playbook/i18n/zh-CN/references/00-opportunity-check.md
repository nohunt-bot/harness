# 前置：机会评估（Opportunity Check）

> **注意：各模式应执行哪些步骤，以 SKILL.md 中「各模式步骤序列」为权威定义。以下仅供参考。**

**适用于：完整模式、高完整性、产出对象为老板/高层**
**可跳过：快速模式、直接实作模式、产出对象为工程师/设计师**

如果使用者是从头开始做 0-to-1 产品，先通过这五个问题。任何一题答案是「否」都是需要重新思考的讯号：

```
| # | 评估问题 | 使用者答案 | 评估 |
|---|----------|-----------|------|
| 1 | 这是否在解决一个真实且急迫的用户痛点？第一批受益的客户是谁？你如何找到他们？ | | ✅/⚠️ |
| 2 | 你是否有解决这个问题的独特优势？目标客户是否至少每周会用到？市场够大吗？ | | ✅/⚠️ |
| 3 | 以现有资源，能否在 2-3 年内做出可用的产品？ | | ✅/⚠️ |
| 4 | 竞争格局如何？你能赢吗？差异化优势是什么？ | | ✅/⚠️ |
| 5 | 是否有可持续的用户增长和商业化路径？ | | ✅/⚠️ |
```

## DHM 初步检验（Gibson Biddle / Netflix）

这个机会是否能做到：
- **D（Delight）**：让用户感到惊喜、超出预期？
- **H（Hard to copy）**：竞争对手难以轻易复制？
- **M（Margin-enhancing）**：随著规模增长，边际利润能提升？

如果某些市场讯号尚不明确，寻找方向性讯号：
- 宏观趋势（例：AI 普及带来的工作流程替代机会）
- 行为转移（用户已在用 workaround，代表需求真实存在）
- 类比市场（找已验证的相似场景）

## 团队热情确认

确认团队对这个问题空间真的有热情。缺乏内在驱动力的团队在寻找 PMF 的路上必然倒下。
