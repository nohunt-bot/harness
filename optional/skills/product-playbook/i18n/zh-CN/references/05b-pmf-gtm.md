# 阶段四：Deliver — PMF + GTM + 商业模式

## 4.3 Todd Jackson（First Round Capital）的四级 PMF 框架

```
| 等级 | 名称 | 特征 | 你的任务 | 典型公司阶段 |
|-----|------|------|---------|------------|
| Level 1 | Nascent PMF（初萌的 PMF） | 找到 3-5 个愿意为解法付费的客户 | 深度服务这些早期用户，找到可重复的模式 | Pre-seed / Seed |
| Level 2 | Developing PMF（发展中的 PMF） | 开始有口碑传播，但增长还不稳定 | 找到可规模化的获客渠道，提升留存 | Seed / Series A |
| Level 3 | Strong PMF（强 PMF）| 「鱼儿自己跳进船里」，转介绍成为主要增长来源 | 建立规模化需求渠道，提升效率指标 | Series A / B |
| Level 4 | Extreme PMF（极端 PMF）| 指标稳健，公司超过 100 人，考虑扩展 TAM | 进入新市场或推出新产品 | Series C+ |
```

**Four P's（在 Level 1-2 阶段找 PMF 的关键四问）：**
1. Problem：你要解决的问题，是否真的足够急迫、足够重要？
2. Promise：你的产品向用户许下了什么承诺？用户相信吗？
3. Product：你的产品是否真的兑现了这个承诺？
4. Pitch：你如何让目标用户发现并相信你的产品？

## 4.7 GTM 策略（Go-to-Market）

**适用于：完整模式 / 高完整性 / 产出对象为老板/业务/行销**

产品做出来之后，如何让目标用户知道并开始使用？

### 获客渠道选择

```
| 渠道类型 | 适合情境 | 典型方法 | 衡量指标 |
|---------|---------|---------|---------|
| 产品驱动增长（PLG） | 用户自助能体验到价值的产品 | 免费试用、Freemium、病毒式邀请 | 注册转换率、启用率、邀请率 |
| 内容与社群 | 目标用户有共同的资讯需求或社群 | 部落格、SEO、社群运营、KOL | 流量、内容转换率、社群活跃度 |
| 付费获客 | 需要快速验证或有明确 ROI | SEM、社群广告、再行销 | CAC、ROAS、LTV/CAC |
| 销售驱动 | B2B 高客单价、需要 demo 和谈判 | SDR、AE、Demo、企业合约 | Pipeline、Close rate、ACV |
| 合作伙伴 | 目标用户集中在某些平台或生态系 | API 整合、分润、共同行销 | 合作渠道转换率、合作伙伴数 |
```

### 初始 100 位用户获取计划

```
目标用户画像：[从 Persona 来]
在哪里找到他们：[他们聚集的线上/线下场所]
用什么 Hook 吸引他们：[从 PR-FAQ 的 Aha Moment 来]
怎么让他们试用：[降低门槛的方法]
初始目标：[X 天内获得 Y 个试用用户]
```

### Launch 策略

| 策略 | 说明 | 适合情境 |
|------|------|---------|
| Closed Beta | 手动邀请 10-50 人，深度收集反馈 | PMF Level 1，产品还在调整 |
| Open Beta | 公开但标明 Beta，降低期待管理 | PMF Level 1-2，需要更多数据 |
| Soft Launch | 在特定市场或渠道低调上线 | 需要真实环境测试但不想全面曝光 |
| Full Launch | 全渠道正式推出 | PMF Level 2-3，产品稳定 |

## 4.8 商业模式与定价

**适用于：完整模式 / 高完整性 / 产出对象为老板/业务**

### 收费模式选择

```
| 模式 | 运作方式 | 适合产品类型 | 关键指标 |
|------|---------|------------|---------|
| Freemium | 基础免费 + 进阶收费 | 用户量大、免费版能展示价值 | 免费→付费转换率（目标 2-5%） |
| Subscription | 按月/年订阅 | 持续提供价值的工具或服务 | MRR/ARR、Churn rate |
| 按用量（Usage-based） | 用多少付多少 | API、云端服务、交易平台 | 每用户平均消费、用量增长率 |
| 一次性付费 | 买断 | 工具型软件、课程 | 客单价、回购率 |
| 交易抽成 | 每笔交易收取比例 | 平台、市场 | GMV、Take rate |
| 按座位（Per-seat） | 依使用人数收费 | B2B 协作工具 | 每帐户平均座位数、扩展率 |
```

### 价值定价对齐

```
定价核心问题：
1. 你的产品帮用户省了/赚了多少？（价值锚定）
2. 用户现在为替代方案付多少钱？（竞争参照）
3. 你的定价是否与用户获得的价值成正比？（公平感）

定价策略：
  - 模式选择：[Freemium / Subscription / 按用量 / ...]
  - 价格区间：[依据价值锚定和竞争参照]
  - 免费层包含什么：[足以到达 Aha Moment，但不足以满足进阶需求]
  - 付费触发点：[什么使用行为或使用量触发升级？]

⚠️ 常见定价错误：
  - 定价太低（传递「不值钱」的讯号）
  - 免费版太强（没有升级动机）
  - 价格和价值不对齐（功能定价 vs 价值定价）
  - B2B 没有年度合约折扣（错失现金流稳定性）
```
