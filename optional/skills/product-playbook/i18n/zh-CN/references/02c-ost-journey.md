# 阶段一：Discovery — OST + Journey Map

## 1.4 机会解法树（Opportunity Solution Tree / OST）

**适用于：完整模式 / 高完整性**

OST 从产品目标出发，系统化链接机会和解法：

```
[产品目标 / Desired Outcome]
    │
    ├── [机会 1：用户痛点或需求]
    │       ├── [解法 1a]
    │       └── [解法 1b]
    ├── [机会 2：用户痛点或需求]
    │       └── [解法 2a]
    └── [机会 3：用户痛点或需求]
            └── [解法 3a]
```

核心原则：
- 目标（Outcome）是可衡量的结果，不是功能或产出
- 机会（Opportunity）来自用户研究，而不是内部脑力激荡
- 解法（Solution）对应机会，不要跳过机会直接讨论解法
- 先广后深：先列出所有机会，再逐一探索解法

## 1.5 User Journey Map

**适用于：完整模式 / 高完整性 / 产出对象为设计师**

**Step 1：概览表**

```
**[Persona名] — 任务：[任务描述]**

| 阶段 | 核心行为 | 情绪 | 关键痛点 |
|---|---|---|---|
| [阶段1] | [一句话描述主要行为] | [情绪 + emoji] | [最重要的那个痛点] |
```

**Step 2：逐段展开细节**

```
> **阶段：[阶段名称]**
> - **Doing（行为）**：[这个阶段用户实际做了什么]
> - **Thinking（想法）**：[用户脑中在想什么，尽量用第一人称口语]
> - **Feeling（感受）**：[情绪状态及原因]
> - **Stakeholder（关系人）**：[这个阶段涉及哪些人]
> - **Problem（痛点）**：[具体的困难或不满]
```

**Step 3：Grouping**
- 如果阶段太细碎，合并成更大的阶段群组
- 统整每个阶段的痛点，标记哪些是核心痛点
